# Autoboot Service Configuration

## Problem Statement

Launch the Heart runtime automatically on Raspberry Pi boot using systemd so installations recover without manual intervention.

## Materials

- Raspberry Pi with sudo access.
- `heart` repository checkout containing `drivers/totem/totem.service`.
- Network access or removable media to transfer files.

## Technical Approach

1. Copy the systemd unit file to `/lib/systemd/system/`.
1. Reload systemd, enable the service, and reboot to verify automatic startup.
1. Use standard systemd tooling to inspect status and logs.

## Reference Guide

Adafruit documentation: <https://learn.adafruit.com/rgb-led-matrix-cube-for-pi/auto-start-on-boot>

## Deployment Steps

```bash
sudo cp heart/drivers/totem/totem.service /lib/systemd/system/totem.service
sudo systemctl daemon-reload
sudo systemctl enable totem.service
sudo reboot
```

## Diagnostics

- Check status: `sudo systemctl status totem.service`
- Review logs: `sudo journalctl -u totem.service`
- Stop service: `sudo systemctl stop totem.service`
- Start service: `sudo systemctl start totem.service`

## Verification

After reboot, confirm that the runtime is active without manual intervention. Inspect `journalctl -u totem.service` if the process fails to start.
